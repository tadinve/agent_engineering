# my-work/

Your own attempts at each class live here — gitignored except this file, so
`git pull`/`git merge` from upstream never conflicts with your in-progress
work.

Structure, one folder per book and class:

```
my-work/
├── book-1/
│   ├── class-04/
│   ├── class-05/
│   └── ...
├── book-2/
│   └── ...
```

See `../HOW-TO-WORK-A-CLASS.md` for the full workflow. Short version: copy
the *previous* class's completed folder from `book-N-.../class-(NN-1)-.../`
in here as your starting point, then follow that class's `BUILD.md` — don't
open the repo's `class-NN-.../` folder until you've made your own attempt,
since that folder already contains the finished answer.
