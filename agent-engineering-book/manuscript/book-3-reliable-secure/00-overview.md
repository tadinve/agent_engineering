# Book 3 — Reliable and Secure Agent Systems
### Resilience, Guardrails and Safe Autonomy

Book 3 turns the capable agent developed in the previous sections into a system that can be trusted under failure, uncertainty and hostile input. Readers study failures originating from models, tools, data, workflows, permissions and human interaction. They then apply established software-engineering patterns — including timeouts, retries, idempotency, circuit breakers and checkpointing — to agentic execution. Security is treated as an enforceable architectural property rather than a warning written inside a prompt. The SDR Lab gains permission boundaries, deterministic guardrails, prompt-injection protections, sandboxed execution and risk-based human approval. By the end of the section, the agent does not merely complete successful runs. It can detect abnormal conditions, preserve completed work, degrade gracefully, escalate appropriately and recover without performing duplicate or unsafe actions.
